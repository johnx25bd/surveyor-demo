# docs

Phase artifacts live here, added as the build progressed. The file number loosely tracks the phase that produced it (build phases 4 and 6 land as code, not docs).

| File | Added in | What it is |
| --- | --- | --- |
| `00-idea.md` | Phase 1 — Idea-gen | The concept brief: what Surveyor is, in plain language. |
| `01-ui-mockup.md`, `01-ui-mockup.html`, `01-ui-mockup.png` | Phase 2 — UI design | The design intent, the interactive mockup it describes, and a screenshot of it. |
| `02-architecture.md` | Phase 3 — Architecture | The binding technical decisions: framework, agent library, interfaces, file layout. |
| `05-phase5-review.md` | Phase 5 — Build phase 2 | The parallel-review findings on the browser build and what's deferred before a public deployment. |

`screenshots/` holds image captures referenced from the docs or the walkthrough. Screen recordings are not committed — they live in the linked YouTube playlist.
